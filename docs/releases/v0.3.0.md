# Verity v0.3.0 — Layer C (Governance)

Release focused on consumer DX, operator integrations, and governance depth.

## Highlights

- **Consumer CLI:** `verity login` (token cache), `verity pull` (API tag resolve + OCI layer download), digest-pin stderr warnings
- **Policy:** `require-digest-on-verify`; `repository-ownership` with optional `verify_with_github_api`
- **Webhooks:** Namespace HTTPS endpoints for `tag.set`, `policy.updated`, `verify.passed`, `verify.failed` (HMAC optional)
- **Docs:** [consumer-getting-started.md](../guides/consumer-getting-started.md), expanded [SECURITY.md](../../SECURITY.md)

## Acceptance criteria

See [mvp-v0.3-release.md](../sdlc/mvp-v0.3-release.md).

## Binaries

- `verity` — publish, inspect, verify, login, pull
- `verity-api` — control plane + webhooks

## Upgrade from v0.2

1. Run database migrations (`VERITY_MIGRATE_ON_STARTUP=true` or your migration job).
2. Configure `VERITY_GITHUB_TOKEN` if using GitHub API ownership checks.
3. Register webhooks: `PUT /v1/namespaces/{ns}/webhooks/{name}`.
4. Share [consumer-getting-started.md](../guides/consumer-getting-started.md) with artifact consumers.

## Known limitations

- CLI interactive OAuth/device flow deferred (env + GitHub Actions OIDC supported)
- Webhooks: at-least-once delivery without operator UI
- CVE blocking, federation: still Deferred
