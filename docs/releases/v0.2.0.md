# Verity v0.2.0 — Layer B (Attribution)

First public release. v0.2.0 is the integrity + attribution baseline (Layers A and B); v0.1.0 was not tagged separately.

## Highlights

- **Sigstore verification:** Local cosign verify on `verity inspect` / `verity verify` (fail-closed when policies require it)
- **Provenance:** SLSA-style attestations, provenance verification on inspect, SBOM attach path
- **Policy:** Push-time and verify-time evaluation for all configured rules (`require-signatures`, `trusted-publishers`, `require-provenance`, `repository-ownership`)
- **GitHub Actions:** `verity-publish` and `verity-verify` composite actions; keyless OIDC golden path
- **Audit API:** `GET /v1/namespaces/{ns}/audit`
- **Commit query:** `GET /v1/namespaces/{ns}/artifacts?commit={sha}`

## Binaries

Release artifacts (Linux tar.gz, Windows zip):

- `verity` — CLI (publish, inspect, verify)
- `verity-api` — API server

## Known limitations

- SLSA Build L1 only; reproducible builds deferred
- SBOM optional on publish
- Repository ownership: provenance + namespace match (no GitHub API in v0.2)
- No `verity pull` / CLI OIDC login yet (Layer C)
- CVE blocking, federation, transparency log: Deferred (#67, #68)
